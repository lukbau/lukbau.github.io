Viewing the first diagram

To view the first diagram you have two option:

  • Viewing a example diagram

  • Viewing a custom diagram

Viewing a example diagram

The OpenNetlistView application provides the possibility to load a example design that is integrated into the program. To do this you need to select one from the Load Example menu which is located inside the File menu. The entries can be seen in Fig. 1

After selecting one of the entries the design is loaded.

Further information about the program can be found in the Further Reading section

Viewing a custom diagram

In order to create a custom diagram the program yosys is required. If it is not installed follow the instructions in the next section, otherwise the section can be skipped.

Installing Yosys

It is recommended to use the oss-cad-suite for installing yosys.

This requires the following steps:

  1. Download the oss-cad-suite, for your system, from the release page on github.: oss-cad-suite

  2. Extract the archive to a directory of your choice.

  3. Add the bin directory to your PATH environment variable.

  4. Run the following command to check if yosys is installed correctly:

yosys -V

Creating the JSON file

Important

The following commands all have the hierarchy -auto-top command in them. This is required in any command used to create a JSON file for OpenNetlistView. Otherwise the program will have problems when determining the top module.

From a verilog file

To create a JSON file from a verilog file you need to run the following command:

For a register transfer level (RTL) file:

yosys -p "read_verilog <verilog_file>; hierarchy -auto-top; prep; write_json <json_file>"

For a gate level file of the ECP5 FPGA:

yosys -p "read_verilog <verilog_file>; hierarchy -auto-top; synth_ecp5; write_json <json_file>"

From a VHDL file

To create a JSON file from a VHDL file you need to run the following command:

For a register transfer level (RTL) file:

yosys -m ghdl -p "ghdl <files> -e <top-module>; hierarchy -auto-top; prep; write_json <json_file>"

For a gate level file of the ECP5 FPGA:

yosys -m ghdl -p "ghdl <files> -e <top-module>; hierarchy -auto-top; synth_ecp5; write_json <json_file>"

Displaying the first Diagram

There are two ways to open the Diagram

  1. Open the program and load the JSON file via the open file dialog. This works for both the native and web assembly version

  2. Start the program with the JSON file as an argument via the command line. This only works on a native version of the application. Further information on this can be found in the chapter Command Line Parameters

Important

When the module is being displayed that is bigger then a certain threshold, a dialog will appear. This is because of the routing library. Which can take a long time to calculate the position of paths for big modules.
